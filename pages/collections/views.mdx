# Views

The <Code>views</Code> section within the <Code>collections</Code> configuration of your admin panel empowers you to control how your collection's data is presented to users. Here's where you can define multiple views, each leveraging different layouts to best suit the type of data and how users might interact with it.

## Example

```js
import { Config, fields, triggers, views } from "@melony/core/config";

export const melonyConfig: Config = {
  ...
  collections: [
    {
      ...
      views: [
        views.kanban({
          slug: "tasksAssignedToMe",
          label: "My Tasks",
          icon: "üìù",
        }),
      ],
    },
  ],
  ...
};
```
